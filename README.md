# 精益项目待办管理系统 (Lean Project Todo Management System)

> 涂装车间精益项目待办事项全流程管理平台 — 纯前端 SPA，无需后端，数据存储在浏览器 localStorage。

## 核心功能

- **多公司多项目管理**：支持多公司隔离，每公司多项目并行管理
- **7 种角色权限体系**：超管、管理员、项目办、项目经理、项目成员、高层、顾问
- **待办全生命周期管理**：新增 → 编辑 → 完成描述 → 审核 → 确认 → 完成/退回/延期
- **项目全景图**：进度概览、里程碑时间线、部门达成率、重要程度分布
- **项目周报**：自动统计 + 手动填写，支持长图导出
- **驾驶舱大屏**：16:9 科技感数据可视化看板（ECharts）
- **审批工作流**：项目结案/删除需高层审批，延期申请需项目经理审批
- **通知系统**：待办审核、周报推送、审批通知、成员邀请
- **移动端适配**：手机/平板自适应，侧边栏抽屉、底部弹出模态框
- **数据导入导出**：支持 Excel (XLSX) 和 JSON 格式

## 技术栈

| 层面 | 技术 |
|------|------|
| 前端框架 | 原生 JavaScript SPA（无框架依赖） |
| 数据存储 | localStorage（key: `lean_mpms_v3`） |
| 图表库 | ECharts v5.5.0（本地 vendor/） |
| 导出库 | SheetJS v0.18.5 / html2canvas v1.4.1（本地 vendor/） |
| 密码加密 | SHA-256 (crypto.subtle + fallback) |

## 数据说明

- 所有数据存储在浏览器 `localStorage`，key 为 `lean_mpms_v3`
- 清除浏览器缓存将重置所有数据
- 支持通过 Excel 导入待办数据（提供模板下载）
- 支持导出单个项目数据为 JSON 或 Excel

## 部署方式

### 方式一：GitHub Pages（推荐）
1. 将代码推送到 GitHub 仓库
2. 在仓库 Settings → Pages 中选择分支和根目录
3. 访问生成的 URL 即可使用

### 方式二：本地直接打开
直接用浏览器打开 `index.html` 文件

### 方式三：静态服务器
```bash
npx serve .
# 或
python -m http.server 8080
```
